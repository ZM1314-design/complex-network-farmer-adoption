# 📖 决策规则详细解释

## 🎯 公式26：决策规则

$$
a_i(t+1) = \begin{cases}
1, & \text{if } P(a_i=1) > \xi \\
0, & \text{otherwise}
\end{cases}
$$

其中：
- $P(a_i=1)$：农户采纳绿色施肥的**概率**（0-1之间，由Logit模型计算得出）
- $\xi \sim \text{Uniform}(0,1)$：一个**随机数**（0-1之间均匀分布）
- $a_i(t+1)$：最终的决策结果（0=不采纳，1=采纳）

---

## 💡 核心思想：用"概率"和"随机数"做决策

### 简单理解

**就像"抛硬币"，但硬币不是公平的：**

1. **Logit模型计算出概率**：例如 `P = 0.65`（65%概率采纳）
2. **生成一个随机数**：例如 `ξ = 0.42`（随机生成，0-1之间）
3. **比较**：
   - 如果 `P > ξ`（0.65 > 0.42）→ **采纳**（a_i = 1）
   - 如果 `P ≤ ξ`（0.65 ≤ 0.42）→ **不采纳**（a_i = 0）

---

## 🎲 具体例子

### 例子1：高概率情况

**场景：**
- Logit计算出：`P(采纳) = 0.85`（85%概率）
- 随机生成：`ξ = 0.32`

**决策过程：**
```
P = 0.85
ξ = 0.32

比较：0.85 > 0.32  ✅

结果：采纳（a_i = 1）
```

**解释：**
- 概率很高（85%），随机数很小（32%）
- 大概率采纳 → **实际采纳了**

---

### 例子2：低概率情况

**场景：**
- Logit计算出：`P(采纳) = 0.15`（15%概率）
- 随机生成：`ξ = 0.78`

**决策过程：**
```
P = 0.15
ξ = 0.78

比较：0.15 > 0.78  ❌

结果：不采纳（a_i = 0）
```

**解释：**
- 概率很低（15%），随机数很大（78%）
- 小概率采纳 → **实际没采纳**

---

### 例子3：临界情况（关键！）

**场景：**
- Logit计算出：`P(采纳) = 0.50`（50%概率，完全随机）
- 随机生成：`ξ = 0.35`

**决策过程：**
```
P = 0.50
ξ = 0.35

比较：0.50 > 0.35  ✅

结果：采纳（a_i = 1）
```

**但如果随机数不同：**
- 如果 `ξ = 0.65`：`0.50 > 0.65` ❌ → **不采纳**
- 如果 `ξ = 0.30`：`0.50 > 0.30` ✅ → **采纳**

**解释：**
- 概率是50%，完全随机
- 随机数决定了最终结果
- 这就是**随机性**的体现！

## 🔄 多次决策的统计规律

### 关键理解：概率的统计意义

**如果同一个农户决策100次：**

**情况1：P = 0.80（80%概率）**
```
决策100次：
- 大约80次采纳（a_i = 1）
- 大约20次不采纳（a_i = 0）
```

**情况2：P = 0.30（30%概率）**
```
决策100次：
- 大约30次采纳（a_i = 1）
- 大约70次不采纳（a_i = 0）
```

**情况3：P = 0.50（50%概率）**
```
决策100次：
- 大约50次采纳（a_i = 1）
- 大约50次不采纳（a_i = 0）
```

**结论：**
- **单次决策是随机的**（由随机数ξ决定）
- **多次决策的统计结果符合概率P**

## 📊 可视化理解

### 概率轴（0-1）

```
0.0 ──────────────────────────────────────────── 1.0
│                                                │
│  ←─────────── P(采纳) = 0.65 ───────────→    │
│                                                │
│  不采纳区域        采纳区域                    │
│  (0 到 0.65)      (0.65 到 1.0)              │
│                                                │
```

**决策规则：**
1. 生成随机数 `ξ`（0-1之间任意位置）
2. 如果 `ξ` 落在"采纳区域"（0到P之间）→ **采纳**
3. 如果 `ξ` 落在"不采纳区域"（P到1之间）→ **不采纳**

**注意：**
- 公式写的是 `P > ξ`，等价于 `ξ < P`
- 意思是：随机数落在0到P之间 → 采纳

## 💻 代码实现

### 公式写法（数学）

```python
if P(a_i=1) > ξ:
    a_i = 1  # 采纳
else:
    a_i = 0  # 不采纳
```

### 代码写法（实际）

```python
# 在 src/dynamics.py 中
prob = self.logit_adoption_probability(idx, subsidy)  # 计算概率P
new_adoptions[idx] = int(np.random.rand() < prob)   # 随机决策
```

**解释：**
- `np.random.rand()`：生成0-1之间的随机数（就是ξ）
- `np.random.rand() < prob`：等价于 `ξ < P`，等价于 `P > ξ`
- `int(...)`：转换为0或1

## 🎯 为什么这样设计？

### 原因1：引入随机性

**真实世界：**
- 即使概率相同，每次决策结果可能不同
- 存在**偶然因素**（情绪、环境、临时因素等）

**模型：**
- 随机数ξ模拟这些偶然因素
- 使得模型更符合真实行为

### 原因2：概率的统计意义

**概率不是"确定性"的：**
- `P = 0.80` 不意味着"80%确定采纳"
- 而是"100次决策中，大约80次采纳"

**决策规则：**
- 单次决策是**随机**的
- 多次决策的**统计结果**符合概率

### 原因3：平滑过渡

**如果不用随机决策（硬阈值）：**
```python
if P > 0.5:
    采纳
else:
    不采纳
```
- 决策是**确定**的（非0即1）
- 在临界点（P=0.5）附近**突变**

**使用随机决策：**
```python
if P > 随机数ξ:
    采纳
```
- 决策是**概率**的
- 在临界区域**平滑过渡**

## 🔍 常见疑问

### 疑问1：为什么是 `P > ξ` 而不是 `P < ξ`？

**答案：**
- 这是**约定**，两种写法都可以
- 关键是：**概率越大，采纳的可能性越大**
- `P > ξ` 意味着：P越大，越容易满足条件，越容易采纳 ✅

**等价写法：**
- `P > ξ`（当前公式）
- `ξ < P`（代码实现）
- 两者完全等价

---

### 疑问2：如果P=0.65，为什么不是"65%确定采纳"？

**答案：**
- 概率不是"确定性"，而是"可能性"
- `P = 0.65` 意味着：
  - **单次决策**：65%的可能性采纳（由随机数决定）
  - **多次决策**：100次中大约65次采纳

**例子：**
```
农户A：P = 0.65
决策10次：
- 第1次：ξ = 0.42 → 0.65 > 0.42 ✅ → 采纳
- 第2次：ξ = 0.78 → 0.65 > 0.78 ❌ → 不采纳
- 第3次：ξ = 0.15 → 0.65 > 0.15 ✅ → 采纳
- ...
- 第10次：ξ = 0.91 → 0.65 > 0.91 ❌ → 不采纳

结果：10次中大约6-7次采纳（符合65%概率）
```

---

### 疑问3：随机数ξ从哪里来？

**答案：**
- `ξ ~ Uniform(0,1)`：从**均匀分布**中随机生成
- 每次决策都生成一个新的随机数
- 模拟真实世界中的**偶然因素**

**代码：**
```python
xi = np.random.rand()  # 生成0-1之间的随机数
# 或者
xi = random.uniform(0, 1)  # 等价
```

## 🎯 总结

### 决策规则的核心

1. **输入**：概率P（0-1之间，由Logit模型计算）
2. **过程**：生成随机数ξ（0-1之间），比较P和ξ
3. **输出**：决策结果（0或1）

### 关键理解

- **单次决策是随机的**（由随机数决定）
- **多次决策的统计结果符合概率P**
- **概率越大，采纳的可能性越大**

### 实际意义

- 模拟真实世界中的**随机性**和**不确定性**
- 使得模型更符合**真实人类行为**
- 在临界区域实现**平滑过渡**

---

**简单记忆：**
- **概率P** = "采纳的可能性"
- **随机数ξ** = "抛硬币的结果"
- **决策规则** = "如果可能性大于硬币结果，就采纳"

**就像"抛一个不公平的硬币"：**
- 硬币正面概率 = P（由Logit计算）
- 抛硬币结果 = ξ（随机生成）
- 如果"正面"（ξ < P）→ 采纳 ✅
