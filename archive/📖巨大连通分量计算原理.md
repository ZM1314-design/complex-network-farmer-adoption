# 📖 巨大连通分量计算原理

## 🎯 核心概念

**巨大连通分量（Giant Connected Component）** 不是"设置"的，而是通过**计算**得到的。它是采纳者子图中最大的连通分量。

---

## 📐 计算步骤

### 步骤1：提取采纳者子图

```python
# 找到所有采纳绿色施肥的农户
adopters = np.where(adoption_states == 1)[0]
# adoption_states = [0, 1, 0, 1, 1, ...]  # 0=传统，1=绿色施肥

# 从完整网络中提取采纳者子图
G_adopters = G.subgraph(adopters).copy()
```

**含义：**
- 从完整的300个农户网络中
- 只保留采纳绿色施肥的农户
- 保留这些农户之间的连接关系
- 形成"采纳者子图"

**示例：**
```
完整网络（300个节点）：
  节点0-299，有连接：0-1, 0-5, 1-3, 2-7, ...

采纳状态：
  节点0: 1（采纳）
  节点1: 1（采纳）
  节点2: 0（不采纳）
  节点3: 1（采纳）
  ...

采纳者子图（只包含采纳的节点）：
  节点0, 1, 3, ...（只保留adoption_state=1的节点）
  连接：0-1（如果节点0和1都采纳，且原网络中有连接）
```

---

### 步骤2：计算所有连通分量

```python
# 计算采纳者子图的所有连通分量
connected_components = list(nx.connected_components(G_adopters))
```

**含义：**
- 连通分量：子图中相互可达的节点集合
- 如果两个采纳者之间有路径连接，它们属于同一个连通分量
- 如果两个采纳者之间没有路径，它们属于不同的连通分量

**示例：**
```
采纳者子图：
  节点：0, 1, 3, 5, 7, 10, 15, 20
  
连接关系：
  0-1, 1-3, 3-5  → 连通分量1: {0, 1, 3, 5}
  7-10, 10-15    → 连通分量2: {7, 10, 15}
  20             → 连通分量3: {20}（孤立节点）

结果：
  connected_components = [
    {0, 1, 3, 5},      # 分量1：4个节点
    {7, 10, 15},       # 分量2：3个节点
    {20}               # 分量3：1个节点
  ]
```

---

### 步骤3：找到最大连通分量（巨大连通分量）

```python
# 找到最大的连通分量
giant_component = max(connected_components, key=len)
giant_size = len(giant_component)
```

**含义：**
- 在所有连通分量中，找到**节点数最多**的那个
- 这就是"巨大连通分量"
- 它的大小 = 包含的节点数

**示例：**
```
连通分量：
  分量1: {0, 1, 3, 5}      → 大小 = 4
  分量2: {7, 10, 15}      → 大小 = 3
  分量3: {20}              → 大小 = 1

巨大连通分量 = 分量1（最大）
giant_size = 4
```

---

### 步骤4：计算相对大小

```python
# 相对大小（相对于总农户数）
total_farmers = len(adoption_states)  # 300
giant_component_ratio = giant_size / total_farmers
```

**含义：**
- 巨大连通分量大小 / 总农户数
- 得到相对大小（0-1之间）
- 表示巨大连通分量占总农户的比例

**示例：**
```
巨大连通分量大小 = 90个节点
总农户数 = 300

巨大连通分量相对大小 = 90 / 300 = 0.30 = 30%
```

## 🔍 完整代码流程

```python
def compute_giant_component_size(self, G: nx.Graph, adoption_states: np.ndarray) -> float:
    """
    计算采纳者子图中的巨大连通分量大小（序参数）
    """
    # 步骤1：找到所有采纳者
    adopters = np.where(adoption_states == 1)[0]
    
    if len(adopters) == 0:
        return 0.0  # 没有采纳者，巨大连通分量大小为0
    
    # 步骤2：创建采纳者子图
    G_adopters = G.subgraph(adopters).copy()
    
    if G_adopters.number_of_nodes() == 0:
        return 0.0
    
    # 步骤3：计算所有连通分量
    connected_components = list(nx.connected_components(G_adopters))
    
    if len(connected_components) == 0:
        return 0.0
    
    # 步骤4：找到最大连通分量（巨大连通分量）
    giant_component = max(connected_components, key=len)
    giant_size = len(giant_component)
    
    # 步骤5：计算相对大小
    total_farmers = len(adoption_states)
    giant_component_ratio = giant_size / total_farmers
    
    return giant_component_ratio
```

## 📊 逾渗相变中的使用

### 在相变分析中

```python
for subsidy in subsidy_range:  # 扫描20个补贴值
    # 运行仿真到收敛
    for _ in range(50):
        dynamics.step_decision_update(subsidy)
        dynamics.update_land_quality(dt=1.0)
    
    # 获取最终采纳状态
    adoption_states = dynamics.farmer_df['adoption_state'].values
    
    # 计算巨大连通分量大小
    giant_size = self.compute_giant_component_size(dynamics.G, adoption_states)
    giant_component_sizes.append(giant_size)
```

**含义：**
- 对每个补贴值，运行仿真得到最终的采纳状态
- 计算该状态下的巨大连通分量大小
- 观察巨大连通分量随补贴的变化

## 🎯 科学意义

### 1. 逾渗相变（Percolation Phase Transition）

**定义：**
- 当巨大连通分量从**接近0**突然跳到**显著值**时，发生逾渗相变
- 表示采纳者从**分散的小簇**变成**连成一片的大簇**

**示例：**
```
补贴 = 1000元：
  采纳率 = 20%
  巨大连通分量 = 5%  （分散的小簇）

补贴 = 1500元：
  采纳率 = 25%
  巨大连通分量 = 8%  （仍然分散）

补贴 = 1800元（临界点）：
  采纳率 = 40%
  巨大连通分量 = 35%  ⭐（突然出现大簇！）

补贴 = 2000元：
  采纳率 = 60%
  巨大连通分量 = 55%  （大簇继续增长）
```

---

### 2. 序参数（Order Parameter）

**巨大连通分量作为序参数：**
- 描述系统从**无序**（分散小簇）到**有序**（连成大簇）的转变
- 在临界点附近发生**突然跳跃**
- 体现**集体行为的涌现**

---

### 3. 临界点识别

```python
# 寻找逾渗临界点：巨大连通分量突然出现的地方
gradient_giant = np.gradient(giant_component_sizes)
percolation_idx = np.argmax(gradient_giant)  # 梯度最大处
percolation_subsidy = subsidy_range[percolation_idx]
percolation_threshold = giant_component_sizes[percolation_idx]
```

**含义：**
- 计算巨大连通分量大小的**梯度**（变化率）
- 梯度最大的地方 = 巨大连通分量**增长最快**的地方
- 这就是**逾渗临界点**

## 🔑 关键点总结

### 1. 巨大连通分量不是"设置"的
- 它是通过**计算**得到的
- 基于实际的采纳状态和网络结构

### 2. 计算过程
1. 提取采纳者子图
2. 计算所有连通分量
3. 找到最大连通分量
4. 计算相对大小

### 3. 科学意义
- **逾渗相变：** 巨大连通分量突然出现
- **序参数：** 描述系统从无序到有序的转变
- **临界点：** 梯度最大处

### 4. 关键参数
- **巨大连通分量大小：** 最大连通分量包含的节点数
- **相对大小：** 巨大连通分量大小 / 总农户数
- **逾渗临界点：** 巨大连通分量突然出现的补贴值

## 💡 实际例子

### 场景1：低补贴（分散状态）

```
补贴 = 500元
采纳率 = 10%
采纳者：30个农户

采纳者子图：
  分量1: {5, 12, 18}        → 3个节点
  分量2: {25, 30}           → 2个节点

  分量4: {80}               → 1个节点
```

**特点：** 采纳者分散成很多小簇，没有大簇

```
巨大连通分量 = 分量1
```

**特点：** 几乎所有采纳者都连成一片
