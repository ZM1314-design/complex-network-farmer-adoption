## 🔄 完整流程逻辑

### 第一阶段：基线模型（Baseline Model）

```
输入数据 → TPB效用函数 → Logit决策 → 基线模型结果
```

#### 详细步骤：

1. **输入数据**
   - 农户属性（经济、教育、风险等）
   - 网络结构（社交关系）
   - 初始状态（5%农户采纳）

2. **TPB效用函数计算**
   ```python
   U_i = θ_p·S_policy + θ_s·S_social - θ_e·C_cost + θ_q·Q_benefit
   ```
   - 计算每个农户的效用值（单位：元）
   - 考虑：政策激励、社会影响、成本、质量收益

3. **Logit决策模型**
   ```python
   P(采纳) = 1 / (1 + exp(-τ·U_i + ε_i))
   ```
   - 将效用值转换为采纳概率
   - 基于概率随机决策

4. **基线模型结果**
   - 每个时间步的采纳率
   - 全局质量演化
   - 系统动力学行为

**特点：**
- ✅ 基于**理论模型**（TPB理论）
- ✅ 决策规则**固定**（Logit模型）
- ✅ 农户**被动响应**（不学习）
- ✅ 结果作为**基准**（Baseline）

---

### 第二阶段：强化学习优化（RL Optimization）

```
相同输入数据 → DQN网络 → 学习最优策略 → 优化结果
```

#### 详细步骤：

1. **相同输入数据**
   - 使用与基线模型相同的农户数据
   - 使用相同的网络结构
   - 相同的初始状态

2. **DQN网络决策**
   ```python
   # 状态向量（12维）
   s_i = [经济, 社会, 政策, 风险, 肥力, 颗粒结构, 生物活性, 
          邻居采纳率, 全局Q, 短视性, 教育优先权, 话语权]
   
   # DQN网络输出
   Q(s_i, a_i) = 神经网络(s_i)
   
   # 选择动作
   a_i = argmax Q(s_i, a)
   ```

3. **学习过程**
   - 通过**试错**学习最优策略
   - 根据**奖励函数**调整决策
   - 考虑**长期收益**（折扣因子）

4. **优化结果**
   - 更高的采纳率
   - 更好的质量提升
   - 更优的长期收益

**特点：**
- ✅ 基于**机器学习**（DQN算法）
- ✅ 决策策略**可学习**（神经网络）
- ✅ 农户**主动学习**（优化策略）
- ✅ 结果作为**优化版本**（RL-optimized）

## 🔍 详细流程对比

### 基线模型决策流程

```
时间步 t:
  ↓
对每个农户 i:
  ↓
1. 计算TPB效用: U_i = compute_utility_tpb(i, action=1, subsidy)
  ↓
2. Logit转换: P_i = 1 / (1 + exp(-τ·U_i + ε_i))
  ↓
3. 随机决策: if random() < P_i: 采纳 else: 不采纳
  ↓
4. 更新状态: adoption_state[i] = 决策结果
  ↓
5. 更新质量: update_land_quality()
```

**代码位置：** `src/dynamics.py` 第360-385行

### 强化学习决策流程

```
Episode e, 时间步 t:
  ↓
对每个农户 i:
  ↓
1. 获取状态: s_i = get_state_vector(i)  # 12维状态向量
  ↓
2. DQN网络: Q(s_i, a) = policy_net(s_i)  # 神经网络输出Q值
  ↓
3. 选择动作: 
   - ε-greedy: if random() < ε: 随机动作
              else: a_i = argmax Q(s_i, a)
  ↓
4. 执行动作: adoption_state[i] = a_i
  ↓
5. 计算奖励: R_i = compute_reward(a_i, ...)
  ↓
6. 观察新状态: s_i' = get_state_vector(i)
  ↓
7. 存储经验: memory.push(s_i, a_i, R_i, s_i', done)
  ↓
8. 学习更新: 
   - 从memory采样batch
   - 计算TD误差
   - 更新网络权重
  ↓
9. 更新质量: update_land_quality()
```

## 💡 关键理解

### 1. 基线模型的作用

**基线模型是"基准"：**
- 展示**理论模型**（TPB）的预测结果
- 作为**对比标准**，评估强化学习的效果
- 代表**传统方法**（基于理论的决策）

**基线模型不学习：**
- 决策规则是**固定的**（Logit模型）
- 农户**被动响应**环境变化
- 无法**优化**长期收益

---

### 2. 强化学习的作用

**强化学习是"优化器"：**
- 通过**试错学习**找到最优策略
- **超越**基线模型的性能
- 考虑**长期收益**（折扣因子）

**强化学习会学习：**
- 决策策略是**可学习的**（神经网络）
- 农户**主动探索**最优策略
- 能够**优化**长期累积奖励

---

### 3. 两者的关系

**不是替代关系，而是对比关系：**

```
基线模型（TPB）         强化学习（DQN）
     ↓                        ↓
   基准性能              优化性能
     ↓                        ↓
        对比分析
             ↓
        性能提升
```

**对比分析：**
- 采纳率提升：RL > Baseline
- 质量提升：RL > Baseline
- 长期收益：RL > Baseline
