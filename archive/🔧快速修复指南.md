# 🔧 快速修复指南

## 问题
错误：`spawn pdflatex ENOENT` - Cursor找不到pdflatex命令

## 解决方案

### 步骤1：找到MiKTeX的实际路径

打开**新的命令行窗口**（不是Cursor的终端），输入：
```bash
where pdflatex
```

这会显示类似：
```
D:\Mitex\miktex\bin\x64\pdflatex.exe
```
或
```
D:\Mitex\miktex\bin\pdflatex.exe
```

**把显示的完整路径复制下来！**

---

### 步骤2：检查路径是否正确

我已经在配置文件中设置了路径：`D:\Mitex\miktex\bin\x64\`

**请检查：**
1. 打开文件资源管理器
2. 进入 `D:\Mitex\miktex\bin\`
3. 看看里面是 `x64` 文件夹还是直接有 `pdflatex.exe`

**如果路径不对，告诉我正确的路径，我帮你修改！**

---

### 步骤3：修改配置文件（如果需要）

如果路径不对，打开 `.vscode/settings.json`，找到：
```json
"command": "D:\\Mitex\\miktex\\bin\\x64\\pdflatex.exe",
```

改为你实际的路径，例如：
- 如果是 `D:\Mitex\miktex\bin\pdflatex.exe`，改为：
  ```json
  "command": "D:\\Mitex\\miktex\\bin\\pdflatex.exe",
  ```

**注意：** 路径中的反斜杠要写成 `\\`（双反斜杠）

---

### 步骤4：重启Cursor

1. **完全关闭Cursor**（所有窗口）
2. **重新打开Cursor**
3. **再次尝试编译**（`Ctrl+Alt+B`）
